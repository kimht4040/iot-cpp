2. 디폴트 값은 선언부분에 위치해야한다.
    - 메인함수 아랫부분에 디폴트 값을 설정할 경우 컴파일러가 이해하지 못함
3. 디폴트 값을 줄 경우 오른쪽부터 채워야한다 
    - 왼쪽부터 줄 경우 컴파일 에러 발생
    - int func(int n = 10, int b ) -- 불가능
    - int func(int a, int b = 10) -- 가능

### namespace [text](iot-cpp/02/untitled/main.cpp)
1. namespace를 사용하여 동일한 이름을 가진 함수를 다른 공간에 분리 가능
    - cout cin endl 등등 기본함수도 std라는 네임스페이스에 존재함
2. namespace {name}으로 선언 하여 사용시 name::(함수 변수명 )으로 사용
3. using namespace std 는 std:: 없이 std라는 이름공간에 접근하겠다는 뜻
    - using namespace를 막 사용할 경우 충돌이 발생할 수 있으니 제한적으로 사용

### 포인터 상수, 상수 포인터 [text](iot-cpp/02/cpp05/main.cpp)
1. 포인터 상수
    - int* const ptr
        - 포인터의 주소 값 자체를 변경 할 수 없음
2. 상수 포인터
    - const int* ptr
        - 포인터가 접근하는 변수의 값을 ptr 로는 변경할 수 없음
3. 상수 포인터 상수
    - const int* const ptr
        - ptr의 값(주소)를 변경할 수 없고 ptr이 접근하는 변수의 값도 변경 불가

3. 생성자 사용시 디폴트 생성자를 사용하고 싶다면 디폴트 생성자를 따로 생성해야함



### 초기화
1. int num = 10;
    - 복사생성자 호출로 초기화
2. int num(10);
    - 직접 초기화로 객체 생성
3. int num{10};
    - 중괄호 직접 초기화
4. int num={10};
    - 복사생성자 호출로 중괄호 초기화

# 교육과정 변경됨

## 클래스
실제로 존재하지 않고 어떤 데이터(속성)를 가지고 어떤 동작(매서드)을 할지 정의 해둔 상태
## 객체
클래스를 바탕으로 메모리에 생성된 실체

### 클래스와 객체의 관계
- 클래스로부터 객체를 만드는 과정을 인스턴스화라고함 이렇게 만들어진 객체를 인스턴스라고 부름(객체 == 인스턴스)
### 생성자
객체가 생성될 경우 가장 먼저 실행되는 매서드 
- 생성자의 종류
    - 디폴드 생성자: 그냥 아무것도 안하고 객체 생성만함
    - 복사 생성자 : 다른객체의 내용을 복사해서 생성함
        - 깊은 복사를 하면(data = new int(*other.data)); 값만 복사하고
        - 얕은 복사를 하면(data = other.data;) 주소를 공유하게됨
            - 얕은 복사를 할 경우 메모리 해제는 한 번 만 해야함
    - 매개변수 생성자: 객체 생성시 원하는 값으로 초기화하여 생성함

### 클래스 다형성
- 상속시 같은 이름의 함수를 다르게 사용
